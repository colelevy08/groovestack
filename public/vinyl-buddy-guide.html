03 — Assembly

Step-by-Step Assembly

Follow these ten steps in order. Each step builds on the last. Test after each major connection.

01
Seat the ESP32 on the Breadboard
Place the ESP32-DevKitC V4 straddling the center channel of your half-size breadboard. Ensure the USB port faces outward for easy access. The module should sit snugly with pins in rows on both sides of the channel.
Tip: Use the bottom half of the board to leave room above for sensor wiring.
02
Wire the Power Rails
Connect the ESP32's 3V3 pin to the breadboard's positive rail and GND to the negative rail. Run a second GND connection from the other side of the ESP32 to the opposite ground rail. All components will share these common rails.
03
Connect the INMP441 Microphone (I2S)
Wire the INMP441 module: WS → GPIO 15, SCK → GPIO 14, SD → GPIO 32. Connect VDD to 3V3 and GND to ground. Tie the L/R pin to GND for left-channel mode.
Tip: Keep the mic wires short and away from the ESP32's antenna to reduce noise.
04
Set Up the I2C Bus (OLED, MPU, RTC, Gesture)
Connect GPIO 21 (SDA) and GPIO 22 (SCL) to a shared I2C bus. Wire the SDA and SCL pins of the SSD1306 OLED, MPU6050, DS3231, and APDS-9960 to this bus. Each device also needs VCC (3V3) and GND. Set the MPU6050 AD0 pin HIGH to shift its address to 0x69, avoiding conflict with the DS3231.
05
Wire the SSD1306 OLED Display
If your OLED has 4 pins (VCC, GND, SCL, SDA), it is already on the I2C bus from Step 4. Position the display facing upward near the top of the breadboard. Confirm address 0x3C by running an I2C scanner sketch.
06
Connect the MicroSD Card Module (SPI)
Wire the SD module: MOSI → GPIO 23, MISO → GPIO 19, CLK → GPIO 18, CS → GPIO 5. Connect VCC to 3V3 and GND. Insert a FAT32-formatted microSD card (up to 32 GB).
07
Attach the WS2812B LED Strip
Connect the LED strip's DIN (data in) to GPIO 4. Power with the 3V3 rail (for 8 LEDs this is fine; for longer strips, use a separate 5V supply). Add a 300-470 ohm resistor in series on the data line for signal integrity.
Tip: A 100 uF capacitor across VCC/GND on the strip prevents voltage spikes.
08
Wire the Buzzer and Button
Connect the piezo buzzer positive lead to GPIO 25 and negative to GND. For the tactile button, wire between GPIO 0 (the BOOT pin) and GND. The internal pull-up resistor on GPIO 0 handles debouncing. Press to trigger identification.
09
Set Up Battery Monitoring
Create a voltage divider with two 100K ohm resistors between the battery positive terminal and GND, with the midpoint connected to GPIO 34. This scales the 4.2V battery voltage down to the safe 0-3.3V ADC range. Connect the 18650 battery to the TP4056 module's B+ and B- pads, and power the ESP32 from the TP4056 output.
10
Test and Enclose
Power on via USB and verify each component: run the I2C scanner, test the mic with a serial audio plot, check LEDs cycle, confirm SD card mounts, and verify the OLED displays the boot screen. Once everything works, transfer to the 3D-printed case. Route wires cleanly and secure the OLED in the case window.
Tip: Flash the firmware (next section) before final assembly to simplify debugging.
04 — Firmware

Firmware & Flashing

Get the Vinyl Buddy firmware running on your ESP32 using the Arduino IDE.

1. Arduino IDE Setup

Install Arduino IDE 2.x and add ESP32 board support:

Arduino IDE → Preferences → Additional Board URLs 
https://raw.githubusercontent.com/espressif/arduino-esp32/gh-pages/package_esp32_index.json

Then go to Tools → Board → Boards Manager, search for esp32 by Espressif, and install the latest version.

2. Required Libraries

Install all of these via Sketch → Include Library → Manage Libraries:

Adafruit_SSD1306 OLED driver
Adafruit_GFX Graphics primitives
Adafruit_NeoPixel WS2812B LEDs
Adafruit_MPU6050 Accelerometer/gyro
SparkFun_APDS9960 Gesture sensor
RTClib DS3231 RTC
ArduinoJson API response parsing
WiFiClientSecure HTTPS for API calls
SD MicroSD file system
driver/i2s.h ESP-IDF I2S (built-in)

3. Board Configuration

Under Tools, set:

Arduino IDE → Tools menu settings
Board:            "ESP32 Dev Module"
Upload Speed:     921600
CPU Frequency:    "240 MHz (WiFi/BT)"
Flash Frequency:  "80 MHz"
Flash Mode:       "QIO"
Flash Size:       "4 MB (32 Mb)"
Partition Scheme: "Default 4MB with spiffs (1.2 MB APP / 1.5 MB SPIFFS)"
PSRAM:            "Disabled"

4. Clone and Flash

Terminal
# Clone the firmware repository
git clone https://github.com/groovestack/vinyl-buddy-firmware.git
cd vinyl-buddy-firmware

# Copy the config template and add your WiFi / API credentials
cp config.example.h config.h
nano config.h
config.h
#define WIFI_SSID       "your-wifi-network"
#define WIFI_PASSWORD   "your-wifi-password"
#define GS_API_KEY      "gs_xxxxxxxxxxxxxxxxxxxxxxxx"
#define GS_DEVICE_ID    "vinyl-buddy-001"
#define GS_API_ENDPOINT "https://api.groovestack.app/v1"

Open vinyl-buddy-firmware.ino in Arduino IDE. Connect the ESP32 via USB, select the correct port under Tools → Port, and click Upload. The OLED should display the GrooveStack logo on first boot.

OTA Updates Supported

After the initial USB flash, future firmware updates can be pushed over Wi-Fi from your GrooveStack dashboard. No need to open the case again.

05 — Help

Troubleshooting FAQ

Common issues and how to resolve them quickly.

Double-check SDA (GPIO 21) and SCL (GPIO 22) connections. Verify the display address is 0x3C using the I2C scanner example sketch (File → Examples → Wire → i2c_scanner). Some OLED modules use 0x3D — update the address in the firmware if needed. Also confirm the display is receiving 3.3V power.

Verify the I2S pin assignments match exactly: WS → GPIO 15, SCK → GPIO 14, SD → GPIO 32. Ensure L/R on the INMP441 is tied to GND. If using a different ESP32 board variant, confirm these GPIOs support I2S peripheral functions. Try swapping the INMP441 module as they can occasionally be DOA.

Hold the BOOT button on the ESP32 while clicking Upload, and release it when the IDE shows "Connecting...". If you have a button wired to GPIO 0, ensure it is not being held low. Also try a different USB cable — some are charge-only and lack data lines. On macOS, you may need the CP2102 or CH340 USB-to-serial driver.

Format the card as FAT32 (not exFAT). Cards larger than 32 GB may need a third-party formatter. Verify SPI wiring: MOSI → GPIO 23, MISO → GPIO 19, CLK → GPIO 18, CS → GPIO 5. Ensure the module is getting stable 3.3V power. Try reducing SPI speed in code to 1 MHz for debugging.

Add a 300-470 ohm resistor on the data line between GPIO 4 and DIN. Place a 100 uF electrolytic capacitor across the strip's VCC and GND. Ensure you are not running the strip at 5V logic while the ESP32 outputs 3.3V — for 8 LEDs at 3.3V this is typically fine, but a logic level shifter may help if colors are unreliable.

The MPU6050 and DS3231 share the default address 0x68. Set the MPU6050's AD0 pin HIGH to shift it to 0x69. Run the I2C scanner to confirm all four devices appear: 0x3C (OLED), 0x39 (APDS-9960), 0x57/0x68 (DS3231), and 0x69 (MPU6050). Add 4.7K ohm pull-up resistors on SDA and SCL if the bus is unreliable.

Verify your API key in config.h is correct and active in your GrooveStack dashboard. Ensure your network allows outbound HTTPS (port 443). The ESP32's WiFiClientSecure library needs a valid root CA certificate or you can set client.setInsecure() for testing. Check serial output for HTTP response codes.

Calibrate the ADC by measuring the actual voltage divider output with a multimeter and adjusting the scaling factor in firmware. The ESP32 ADC is non-linear — the firmware includes a calibration lookup table. Use GPIO 34 specifically, as it is input-only and does not have internal pull-ups that could affect readings. Ensure both resistors in the divider are closely matched.
